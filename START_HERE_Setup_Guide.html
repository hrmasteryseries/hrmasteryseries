Put hrmasteryseries Live — Step-by-Step Guide

For: Mayur N Koikar  |  Time needed: about 30–40 minutes, once.
Cost: ₹0 — GitHub hosting is free forever for sites like this.

You will do 4 things, in order: (1) create a free GitHub account, (2) upload the website folder, (3) switch the website on, (4) connect your GoDaddy domain. Every click is written out below. You cannot break anything — every step can be undone.

Step 1 — Create a free GitHub account (5 min)

  1. Go to github.com/signup in your browser.
  2. Enter your email (mayurkoikar@gmail.com), create a password, and choose a username — suggestion: mayurkoikar or hrmasteryseries. Write the username down; you need it in Step 4.
  3. Verify the email code GitHub sends you. Choose the Free plan if asked. Done.

Step 2 — Upload the website (10 min)

  1. Once signed in, go to github.com/new (this creates a "repository" — just a folder on GitHub).
  2. Repository name: type exactly hrmasteryseries. Set it to Public. Do NOT tick any other boxes. Click the green Create repository button.
  3. On the next page, click the small link "uploading an existing file".
  4. On your computer, open the folder hrmasteryseries-website (it is inside your "Raw Books for Audit" folder). Press Ctrl+A to select everything inside it, and drag it all onto the GitHub upload page. Important: drag the contents of the folder, not the folder itself — you should see index.html, books.html, the css folder, the covers folder, etc. appear in the upload list.
  5. Wait for all files to finish uploading, then click the green Commit changes button at the bottom.
If the drag-and-drop misses the sub-folders (css, js, data, covers, resources), upload them one folder at a time: click Add file → Upload files, then drag one folder in, commit, repeat.

Step 3 — Switch the website on (2 min)

  1. In your repository, click Settings (top menu), then Pages (left sidebar).
  2. Under Branch, choose main and folder / (root), then click Save.
  3. Wait 2–3 minutes and refresh the page. GitHub will show: "Your site is live at https://YOUR-USERNAME.github.io/hrmasteryseries/". Click it — your website is on the internet! Check it works before continuing.

Step 4 — Connect your GoDaddy domain (10 min)

Still in Settings → Pages:

  1. Under Custom domain, type your domain (e.g., hrmasteryseries.com — use whatever ending you bought: .com, .in, etc.) and click Save. GitHub may show a warning — that's expected until GoDaddy is set up.
  2. Now open a new tab: go to godaddy.com → sign in → My Products → find your domain → click DNS (or "Manage DNS").
  3. Delete any existing records of type A with name @ (including "Parked"). Then add these 4 new A records (click "Add New Record" each time):
    TypeNameValueTTL
    A@185.199.108.153Default / 1 hour
    A@185.199.109.153Default / 1 hour
    A@185.199.110.153Default / 1 hour
    A@185.199.111.153Default / 1 hour
  4. Add 1 more record of type CNAME: Name = www, Value = YOUR-USERNAME.github.io (your GitHub username from Step 1, followed by .github.io — for example mayurkoikar.github.io). If a www CNAME already exists, edit it to this value.
  5. Save everything. Wait 30 minutes to a few hours (GoDaddy can be slow). Then visit your domain — your website will be live on it.
  6. Final touch: back in GitHub Settings → Pages, tick Enforce HTTPS once the option becomes clickable (may take an hour). This gives you the secure padlock.

How to add a new book later (5 min, any time)

  1. Open your website and add /admin.html to the address — e.g., hrmasteryseries.com/admin.html. This is your private control page (it is not linked anywhere on the site).
  2. Click + Add a new book, fill in the name, cover, description, sample chapter, and your Amazon / Pothi / other links. Click Save this book.
  3. In section 3 of that page, click Download updated data.js.
  4. Go to your GitHub repository → open the data folder → Add file → Upload files → drop in the downloaded data.js → Commit changes. Your site updates in about a minute.
Adding a cover image or a free PDF resource: same idea — in GitHub open the covers folder (for cover images) or resources folder (for PDFs), click Add file → Upload files, drop the file in, commit. Then reference it in the admin page as covers/yourfile.jpg or resources/yourfile.pdf.

If anything goes wrong

Nothing here is fragile. If the site looks broken after an upload, tell Claude what you did and what you see — it is always fixable, usually by re-uploading one file. Your books and files on your own computer are never touched by any of this.
